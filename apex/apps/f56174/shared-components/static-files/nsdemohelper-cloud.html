How NetSuite Demo Helper Works

The tool detects the local Codex app and uses Codex background operators to turn company context, discovery notes, audience choices, demo scope, and strategy into practical SC demo assets.

Current version: Oracle APEX cloud workspace with optional local Codex bridge. Target version: standalone NSDemoHelper app for Mac and Windows, with update checks from GitHub releases.

1. BriefAdd the company site, demo request, discovery notes, and SC instructions.
2. ShapeChoose audience type, target segment, strategy, industry, language, and story mode.
3. LearnCodex reviews the context and infers likely ERP priorities and demo pressure points.
4. GenerateCodex authors the SC guide; the helper turns it into an editable manifest with navigation, narration, proof points, and safe actions.
5. CheckCodex reviews the guide and manifest; the Intelligence tab structures the risks, gaps, pacing, stakeholders, and winning moments.
6. CoachThe SC guide gives the demo story, talk track, setup prompt, and asset prompt.
7. RehearseRehearsal checks routes, buffers account prep, captures timing, and warms the flow.
8. RunThe live demo drives NetSuite and narrates with the selected voice engine.
